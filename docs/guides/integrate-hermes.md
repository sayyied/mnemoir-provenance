# Integrate the Hermes reference adapter

Hermes integration is optional and explicit. Mnemoir and Hermes must be importable in the same Python runtime. In a new shared environment run `python -m pip install 'mnemoir-provenance[hermes]'`; for existing Hermes, run that environment's Python with `-m pip install 'mnemoir-provenance[hermes]'`. If Mnemoir is already installed in that runtime and the intent is to update it, add `--upgrade`; do not add it merely for a first installation or an exact version pin. Then run `mnemoir plugin install --hermes-home <HOME>` and `mnemoir plugin status --hermes-home <HOME> --hermes-python <HERMES_PYTHON>`. The installer copies the provider and creates only its default storage parent at mode 0700. It never edits config, selects a provider, restarts a gateway, ingests, promotes, or writes back.

Selection remains separate: `hermes memory setup mnemoir_provenance`, followed by `hermes memory status`. Memory status is authoritative; `hermes plugins list` describes general plugin enablement and may disagree without meaning selection failed. Start a new process/session through the operator's normal procedure after selection. Fresh recall is expected to be empty/degraded because `ingest_on_start=false`. Use `mnemoir plugin bootstrap-profile --hermes-home <HOME> --profile-root <CONTROLLED_ROOT> --profile-id <ID> --verify-query <TEXT>` for a bounded first source. It accepts only immediate non-symlink MEMORY.md/USER.md, emits the closed bootstrap schema, is idempotent, and never promotes or writes back. A `bootstrap_no_cited_match` keeps committed evidence; replay with a matching query.

Disable with `hermes memory off`, verify with `hermes memory status`, and retain/backup the SQLite DB separately. Removing the plugin does not delete data. The historical private provider `council_memory_core` and public provider `mnemoir_provenance` are separate: no automatic selection rewrite or DB migration occurs.
