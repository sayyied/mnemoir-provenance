# Python API reference

Stable entry points are organized by module: `db`, `sources`, `ingest`, `recall`, `retrieval`, `curation`, `scoring`, `council`, `autonomy`, `source_adapters`, `overflow_policy`, `live_overflow`, `operator_api`, `local_ui`, and `wiki_projection`.

Treat functions prefixed with `_`, benchmark-provider helpers, migration helpers, and experiment internals as unstable. Pin the candidate version and test exact error codes.
